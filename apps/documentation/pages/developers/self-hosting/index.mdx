# Getting Started with Self-Hosting

This is a step-by-step guide to setting up and hosting your own ShieldSign instance.

## Documentation

- [How to Self-Host](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/self-hosting/how-to.mdx) - Complete deployment guide with Docker and manual setup
- [Signing Certificate](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/self-hosting/signing-certificate.mdx) - Generate or purchase a signing certificate
- [OAuth Providers](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/self-hosting/setting-up-oauth-providers.mdx) - Configure Google, Microsoft, and OIDC authentication
- [Telemetry](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/self-hosting/telemetry.mdx) - Understanding and disabling telemetry

## Quick Start

1. Clone the repository
2. Configure environment variables
3. Set up your signing certificate
4. Deploy with Docker or build from source

See the [How to Self-Host](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/self-hosting/how-to.mdx) guide for detailed instructions.
