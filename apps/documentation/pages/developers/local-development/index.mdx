# Local development

There are multiple ways of setting up ShieldSign for local development. At the moment of writing this documentation, there are 3 ways of running ShieldSign locally:

- [Using the developer quickstart with Docker](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/local-development/quickstart.mdx)
- [Manually setting up the development environment](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/local-development/manual.mdx)
- [Using Gitpod](https://github.com/KatalystDigital/shieldsign-core/blob/main/apps/documentation/pages/developers/local-development/gitpod.mdx)

Pick the one that fits your needs the best.

## Tech Stack

- [Typescript](https://www.typescriptlang.org/) - Language
- [React Router](https://reactrouter.com/) - Framework
- [Prisma](https://www.prisma.io/) - ORM
- [Tailwind](https://tailwindcss.com/) - CSS
- [shadcn/ui](https://ui.shadcn.com/) - Component Library
- [react-email](https://react.email/) - Email Templates
- [tRPC](https://trpc.io/) - API
- [React-PDF](https://github.com/wojtekmaj/react-pdf) - Viewing PDFs
- [PDF-Lib](https://github.com/Hopding/pdf-lib) - PDF manipulation
- [Stripe](https://stripe.com/) - Payments
